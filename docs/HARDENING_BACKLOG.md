# Hardening Backlog

This file tracks security, release, and production-hardening work intentionally
deferred while the current v2-mixed pass focuses on functional completeness and
operator workflow polish. These items are not waived; they remain required for a
later hardening pass.

Status values such as `Addressed`, `Functional`, or `implemented` describe the
bounded functional slice recorded in the row. They do not close the production
hardening requirement unless the same status also says the production custody,
field-evidence, scale, or certification requirement is complete.
