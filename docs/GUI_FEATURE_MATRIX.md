## Reading Notes

Confidence levels:

- High: accessible official docs support the feature claim.
- Medium: accessible docs support adjacent behavior, but the exact GUI pattern needs manual confirmation.
- Low: public pages were blocked, unstable, or less specific during this research pass.

Best-of-breed nominations require citations. User pain signal is not treated as a factual product claim.

## Current Implementation Snapshot

The vendor and domain matrices below remain the 2026-06-17 research baseline
and citations are intentionally preserved. They should not be read as saying
the WebUI is still research-only. The current `v2-mixed` WebUI has broad route
coverage, with the latest remote `make webui-enterprise-smoke` pass covering
the current 20-route canonical enterprise smoke set, including `/proxy` and
`/compliance`, across desktop, tablet, and mobile. The Makefile enterprise path
list also accepts `/dashboard` as an alias for `/`, so the configured path list
has one more entry than the canonical route count. That pass is continuation
evidence for the current worktree, not durable release evidence; release
evidence still needs to be regenerated through the repo-local release tooling
after source-control acceptance.
